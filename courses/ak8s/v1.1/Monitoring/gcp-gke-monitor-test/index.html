Custom Metrics

The Custom Metrics function will increment a counter which is then exposed to Stackdriver Monitoring. In this example, the counter represents the number of active users in an application. You can increase or decrease the number of active users; however the value is only reported to Stackdriver once per minute .

Click the Start Monitoring button to create the Stackdriver metric descriptor. This button will also trigger the test utility to start exporting the timeSeries data (the custom metric values) to Stackdriver.

Crash the Pod

This button will invoke a function that has an unhandled error, which causes the pod to crash and restart.

When you click the button, your browser will return an error. Wait a 30 seconds while Kubernetes spawns a replacement container, then refresh the page.
